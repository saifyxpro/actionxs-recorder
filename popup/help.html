Recording Status Indicators

Recording

Active Recording: All interactions are being captured. The red pulsing indicator shows active recording state.

WAIT

Page Loading: Extension is waiting for page navigation to complete. Do not perform actions during this state.

Paused

Recording Paused: Interactions are temporarily ignored. Resume when ready to continue recording.

‚ö†Ô∏è Important: After page navigation, always wait for the status to change from WAIT to Recording before continuing with interactions.

üéõÔ∏è Recording Controls Reference

Start Recording Ctrl+R

Begins a new recording session. Clears any previous recordings and starts capturing interactions.

Pause Space

Temporarily pauses recording. Use when you need to perform actions that shouldn't be included in automation (like opening developer tools).

Resume Space

Resumes recording after being paused. Button automatically changes from "Pause" to "Resume" based on current state.

Stop Ctrl+S

Ends the current recording session and makes recorded actions available for export and review.

Export Ctrl+E

Copies all recorded actions to clipboard in JSON format, ready for importing into AdsPower RPA workflow.

Restart Ctrl+Shift+R

Clears all recorded actions and starts fresh. This action cannot be undone, so use with caution.

üîß Troubleshooting Common Issues

Recording doesn't start or stops unexpectedly

Possible causes and solutions:

  • Refresh the current page and try again
  • Check if the page URL is supported (avoid chrome:// pages)
  • Ensure the extension has proper permissions
  • Try disabling other Chrome extensions temporarily
  • Restart the browser if the issue persists
Some clicks or inputs are not recorded

Common reasons and fixes:

  • The element might be inside an iframe (limited support)
  • The element uses complex event handling
  • Recording was paused during the action
  • The action happened during page navigation (WAIT state)
  • Try clicking more slowly and deliberately
Export button doesn't work

Clipboard access issues:

  • Grant clipboard permissions to the extension
  • Try clicking the export button again
  • Check if other applications are using the clipboard
  • Restart the browser and try again
Recorded selectors don't work in AdsPower

Element selector issues:

  • The page structure might have changed
  • Elements might be dynamically generated
  • Try recording the action again more precisely
  • Use browser developer tools to verify selectors
  • Consider using more stable element attributes
